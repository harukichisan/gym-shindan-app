# ジム診断ブラウザアプリ UI/UX設計書（v0.1）

更新日: 2026-01-29  
根拠: `docs/requirements.md` v0.2

## 1. 目的
- 初心者が短時間で「自分に合いそうなジムタイプ」を理解できる
- 不安/痛み配慮を明確にし、安心して次アクションへ進める
- 根拠・注意・チェックリストにより専門性と信頼感を担保する

## 2. 方向性（合意）
- デザイン方向: **フレンドリー×専門的**（Headspace寄り）＋**やさしい質感**（Calm寄り）
- 価値観: 安心感 ＞ 分かりやすさ ＞ 続けやすさ
- MVP機能: **共有機能のみ**（保存/履歴は後回し）

## 3. トーン&コピー
- 断定を避ける: 「合いやすい」「傾向」「可能性」
- 不安に寄り添う: 「迷って当然」「初めてでも大丈夫」
- 安全配慮: 痛み/不調は医療判断ではないことを明記

## 4. ビジュアルシステム
### 4.1 カラー（初稿）
- Base / Primary: セージグリーン `#7BAE9E`
- CTA Accent: コーラル `#F28C7A`
- Info Accent: スカイ `#8FBCE6`
- Warning (soft): ダスティレッド `#D77474`
- Background: クリーム `#F7F3ED` → 薄セージ `#E8F1EE`

### 4.2 タイポグラフィ
- 見出し: Zen Maru Gothic（丸みで安心感）
- 本文: Zen Kaku Gothic New（読みやすく専門感）
- 数値/タグ: IBM Plex Sans JP（視認性）

### 4.3 形状・影
- 角丸: 16–20px
- 影: ぼかし大きめ・弱い不透明度（柔らかい浮遊感）

### 4.4 イメージ
- 写真は使わない（心理的ハードル低減）
- ゆるい線画アイコン中心（痛み/不安の話題でも穏やかに見せる）

## 5. 情報設計・導線
### 5.1 全体フロー
Start → Question →（Pain/不調分岐）→ Review（任意）→ Result

### 5.2 共通原則
- 1画面1問
- 進捗と現在セクションを上部固定
- 「戻る」「次へ」を下部固定
- 迷いが出やすい質問には補助チップを表示

## 6. 画面設計
### 6.1 Start
- タイトル: 「3分で、あなたに合いやすいジムの“種類”がわかります」
- モード選択カード
  - 簡単診断: 所要時間/質問数バッジ
  - 複雑診断: 痛み/不調の配慮ありを明記
- 免責の同意チェック（必須）
- CTA: 「診断をはじめる」

### 6.2 Question
- 上部: 進捗バー＋セクション名
- 中央: 質問文＋選択肢（例文つき）
- 下部: 戻る/次へ（固定）
- 補助: 「迷ったらこれ」チップ

### 6.3 痛み/不調セクション導入
- 注意ボックス（医療判断でない旨）
- 「続ける」「戻る」

### 6.4 Result
上から順に表示する。  
1) 注意表示（痛み/不調がある場合は固定表示）  
2) 「あなたに合いやすい順」上位3カード  
3) 各カード詳細（合う理由/注意点/料金目安）  
4) 体験時チェックリスト（アコーディオン）  
5) 継続ボトルネック  
6) 4週間ミニプラン  
7) 施設に伝えるテンプレ  
8) 共有ボタン（リンクコピー）

## 7. コンポーネント仕様（概要）
- モードカード: 所要時間バッジ／説明短文／選択時ハイライト
- 選択肢カード: タップ領域44px以上／選択時に色面＋アイコン
- 進捗バー: セクション色が薄く切り替わる
- 注意ボックス: 情報/注意/強注意の3段階
- 結果カード: 順位ラベル＋根拠箇条書き
- チェックリスト: 5–10項目、折りたたみ
- 共有: 「リンクをコピー」＋「個人情報は含みません」

## 8. 動き
- 画面遷移: フェード＋軽いスライド（200–250ms）
- 結果カード: 200msずつ段階的に表示
- 進捗バー: 滑らかに伸長

## 9. アクセシビリティ
- コントラスト比 4.5:1 以上
- タップ領域 44px 以上
- 文字サイズはOS拡大に追従
- 長文はアコーディオンで負荷軽減

## 10. 共有機能（MVP）
- URLパラメータで結果再現（個人情報なし）
- 共有画面に「個人情報は含みません」を明記
- SNSプレビューはトップ1タイプのみ表示（過度な個人性回避）
